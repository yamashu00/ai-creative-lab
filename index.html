What is "Vibe Coding"?

文法暗記はもう終わり。AIと共に「創る」ことに集中する、新しい開発スタイル。

1. 共同的なアプローチ (Co-creation)

AIは「魔法の杖」ではなく「相棒」

一発で完璧なアプリが出てくることは稀です。大切なのは、AIと対話（キャッチボール）をすること。
人間がアイデアを出し、AIが実装し、人間が確認して、またAIに修正を頼む。このサイクルの繰り返しこそが開発です。

POINT:
「作って終わり」ではありません。AIが出してきたものに対して「もっとこうして」「ここは違う」とフィードバックを返すことで、どんどん良くなっていきます。

「英単語アプリを作って」
（コードを生成...）「作りました！」
「ありがとう！でも文字が小さくて見にくいから、もっと大きくして。あと背景は黒がいいな」
↑ このやり取りがVibe Coding！

2. 抽象度のコントロール (Abstraction)

AIへの指示には「抽象度（具体性のレベル）」があります。これを使い分けるのが上級者への近道です。 エレベーターのように、状況に合わせて階層を行き来しましょう。

高層階（抽象度：高）

企画・デザイン段階

ふわっとした指示。全体の雰囲気やコンセプトを伝える時。

"近未来的でサイバーパンクな雰囲気にして"
"高校生が毎日使いたくなるような楽しい感じで"

低層階（抽象度：低）

修正・デバッグ段階

カチッとした指示。具体的な動作や数値、エラー修正をする時。

"背景色は #0f172a に変更して"
"ボタンを押した時に alert('OK') を出して"

いつ、どのレベルで話す？

最初 (Start)
抽象的
中盤 (Dev)
Mix
仕上げ (Fix)
具体的

最初は「ふわっと」作り始めて、徐々に「細かく」指示していくのがコツです。

3. 段階的な開発 (Step-by-Step)

いきなり完成品を作ろうとすると失敗します。Webアプリ開発は「積み木」のようなものです。
まずは土台を作り、少しずつ機能を足していく「スモールスタート」を心がけましょう。

Level 1

動くゴミ (MVP)

デザインは適当でいい。とにかくボタンを押したら反応するだけの、最小限のものを作る。

Level 2

見た目を整える

Tailwind CSSなどを適用して、それっぽくする。「抽象度：高」の指示で雰囲気を変える。

Level 3

機能追加・保存

localStorageでデータを保存したり、音を出したり、複雑な機能を追加して完成度を高める。

Project Roadmap

3学期を通して、アイデアを形にしていくスケジュールです。

Phase 0: ハンズオン体験

Day 1

まずは理屈より実践。簡単なアプリをAIと一緒に作り、完成までの一連の流れ（生成→実行→公開）を体験します。

Phase 1: 企画 & AIブレスト

1-2週目

専用のAIアシスタントと会話しながら、「どんな機能が必要か」「どんなデザインにするか」を整理し、企画を固めます。

Goal: 「こんなアプリを作る！」という企画書が完成していること。
アイデア発想Gemを開く

Phase 2: 実装 & プレビュー

3-4週目

Canvas機能に特化したAIを使ってコードを生成します。AIに指示を出し、リアルタイムでコードが出来上がっていく様子を体験しましょう。

Goal: AIを使ってアプリの基本コードを生成すること。
Canvas機能Gemを開く

Phase 3: プロトタイピング

5週目

Gemini Canvasのプレビュー機能を使って、その場で動作確認を行います。PCにファイルを保存する必要はありません。動かない場合はすぐにAIに修正を依頼しましょう。

Goal: Canvas上のプレビュー画面で、アプリが正しく動く状態にすること。

Phase 4: 公開

6週目

完成したコードをGASに移してWeb公開します。世界に一つだけのURLが発行されます。

Goal: デプロイURLを発行し、提出すること。
提出フォームを開く

Phase 5: 相互評価

7週目

クラスメイトにURLを共有して遊んでもらいましょう。良い点や改善点をフィードバックし合い、さらなるブラッシュアップにつなげます。

Goal: お互いの作品をリスペクトし、評価フォームに入力すること。

Geminiの画面左上にあるモデル選択メニューを確認してください。

生成 (Generate)

Geminiにコードを書いてもらいます。

基本のプロンプト

以下のプロンプトをコピーして、Geminiに入力（Cmd + V）してください。

あなたは優秀なWebエンジニアです。高校生向けの学習用Webアプリを作りたいです。 【テーマ】 シンプルな「ストップウォッチ」 【要件】 1. デザインはTailwind CSSを使用し、黒背景にネオンカラー（緑やピンク）を使ったサイバーなデザインにしてください。 2. スタート、ストップ、リセットボタンをつけてください。 3. 時間は「00:00:00」形式で大きく表示してください。 コードのみを出力してください。
03

構築 (Build)

Google Apps Scriptでアプリを作成します。

GASプロジェクトの作成

1. プロジェクト作成

Apps Scriptのホーム へ行き、「新しいプロジェクト」をクリック。

2. コード.gs の編集

元々入っているコードをクリックし、全選択（Cmd + A）してDeleteキーで削除。その後、以下を貼り付け（Cmd + V）ます。※これがないと動きません。

function doGet() { return HtmlService.createHtmlOutputFromFile('index') .addMetaTag('viewport', 'width=device-width, initial-scale=1'); }

3. HTMLファイルの作成

画面左上の「ファイル +」ボタン > 「HTML」を選択 > 名前を「index」にする。
Geminiが生成したコードをコピーし、ここに貼り付け（Cmd + V）ます。

04

公開 (Deploy)

アプリをインターネットに公開します。

デプロイの手順（重要）

一つでも設定を間違えると動かないので、慎重に進めてください。

  1. 画面右上の「デプロイ」ボタンをクリック
    出てきたメニューから「新しいデプロイ」を選択します。
  2. 種類の選択
    左側の歯車アイコン⚙️をクリックし、「ウェブアプリ」を選びます。
  3. 設定項目の入力
    ・説明: （空欄でもOK）
    ・次のユーザーとして実行: 自分
    ・アクセスできるユーザー: 聖学院中学校・高等学校 内の全員

    ※「自分のみ」にすると他の人が見られません。

  4. デプロイ実行
    右下の「デプロイ」ボタンを押します。
    ※初回は「アクセスを承認」などの画面が出ることがあります。その場合は指示に従って許可してください。
  5. 完了
    「ウェブアプリ」の下に表示されたURLが、あなたのアプリのURLです！クリックして開いてみましょう。

AIとの新しい向き合い方

"Softening the language"（表現を柔軟にする）

最新の生成AIは、非常に賢くなっています。以前のように「絶対に〜して」「〜以外は禁止」とガチガチに命令するよりも、AIに考える余地（余白）を与える方が、良い結果が出ることがわかってきました。

Old Style

「コードだけを出力しろ。説明は一切不要。絶対にCSSは別ファイルにするな。」

→ AIの思考を狭めてしまい、創造性が下がるかも。

New Style

「HTML1ファイルにまとめると管理しやすいです。デザインも見やすく調整してください。」

→ 目的を伝えて、手段はAIの提案力に任せる。

Prompt Library

もっとカッコよくしたい

今のデザインは少し地味です。高校生が好むような、サイバーパンク風のデザインに変更してください。ボタンはネオンのように光らせて、ホバーすると色が変化するアニメーションをつけてください。

データを保存したい

今のままだとリロードするとデータが消えてしまいます。JavaScriptのlocalStorageを使って、入力した内容やスコアをブラウザに保存し、次回開いたときも残っているように修正してください。

エラーで動かない

コードを実行したら動きません。コンソールに以下のエラーが出ています。
[ここにエラーメッセージを貼り付け]
この原因を特定し、修正したコード全体を再度出力してください。

画像が出ない (GAS用)

Google Apps Scriptで動かしたら画像が表示されません（リンク切れマークになります）。
相対パスを使わず、画像データをBase64エンコードしてimgタグのsrcに直接埋め込む形式に書き換えてください。
